Refined Attack Model (use this exact definition)
TCP Retransmission Smuggling exploits inconsistencies in how stateful network inspectors (WAFs, IDS/IPS) handle TCP retransmissions during stream reassembly, compared to the actual backend application.
Core primitive:

Attacker sends a TCP segment containing a payload.
The inspector processes the initial transmission (normalizes, applies rules, deduplicates, etc.).
Due to natural or induced packet loss, TCP retransmits the segment.
The inspector’s reassembly engine treats the retransmitted segment asymmetrically (skips re-inspection, applies different normalization, fails to deduplicate correctly, or has a different view of “already seen” bytes).
The backend receives the full stream and interprets the payload differently — enabling filter bypass, injection, or evasion that would have been blocked on the first pass.

Key insight: TCP retransmission is normally invisible above L4. Any asymmetry in how an inspector handles the retransmit event creates a desync with the endpoint — a TCP-layer analog to HTTP Request Smuggling.
Target surface (priority order):

Open-source IDS/IPS stream reassemblers (Zeek, Suricata, Snort)
WAF stream handling (ModSecurity, custom proxies)
Any system that performs TCP reassembly + content inspection

Not in scope for this research:

Pure endpoint charset flipping (unrealistic — apps don’t see retransmit flags)
Off-path forced retransmission (too unreliable)
Production exploitation

Research Questions to Answer

Do Zeek or Suricata exhibit asymmetric behavior between original and retransmitted TCP segments during reassembly?
Can we construct a payload + retransmit pattern that causes an inspector to either:
Skip inspection on the retransmit, or
Normalize the data differently than the backend?

Under what conditions (natural loss, spurious retransmit, overlap, specific sequence numbers) does desync occur?
What is the practical impact on common filter types (string matching, regex, encoding normalization, etc.)?
How can defenders detect or mitigate this class of evasion?
